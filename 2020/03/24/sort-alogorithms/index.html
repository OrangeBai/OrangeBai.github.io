Basic Sorts

Mar 24, 2020 in Alogorithms

Intro

Sorting problems are the most basic problems one might encounter during coding. Here I would like to list the most basic sorting algorithms, with pseudocode and discussion. All the pics are extracted from Introduction to Algorithms [1]. C ++ code and Python code will be published at my GitHub in the following days.

Backgroud

pic1 here

A practical scenario for a sorting problem is soring a hand of cards as figure 1 illustrated. Intuitively, when a new card is dealt with you, you might wanna compare it with the other cards in your hand, if they are sorted, and find the most suitable position for it, which is exactly what an insertion sort is doing. However, thought it might be the most easy algorithm to understand, insertion algorithm lacks efficiency. With an average running time of $\Theta(n^2)$, it is slower than most of the sorting algorithms. The way we used for insertion sort is known as incremental approach, which lets the inputs come one by one and have them meet the requirements.

Another algorithm design technique that widely applied in sorting problems and other problems is divide and conquer. It breaks down the problem into sub-problems recursively, until they can be solved directly. The solution to the sub-problems is then combined to give a solution to the origin problem. Quicksort, which is the built-in sort function in most languages and considered as the fastest sorting algorithm, is an example of that.

Loop invariants

Before start, we would like to introduce the concept of Loop invariants. Similar to mathematical induction, if a loop algorithms has the following properties:

  1. Initializatin: It is true prior to the first iteration of the loop.
  2. Maintenance: If it is true before an iteration of loop, it remains true before next iteration.
  3. Termination: When the loop terminates, the invariant gives us a useful property that helps show that the algorithm is correct.

It is similar to mathematical induction in some sense.

Algorithms

Insertion sort

Insertion sort is one of the easiest algorithms to understand. It iters from one side of the array to the other side and find a suitable position for the new added value. Just like when a new card is dealt in a card game, if all the other cards are sorted, the player can start from one side and compare it with the new one until a suitable position is found.

Discription and Correctness

  1. Initializatin: At the very begining of the sort when $j = 2$, $A[1,…,j-1]$ only contains one element and can be regarded as sorted array.
  2. Matintenance: It can be asuumed that at the begining of each loop, $A[1,…,j-1]$ is sorted. During the loop, $A[j]$ is compared with $A[j-1], A[j-2]…A[1]$ one by one until find its suitable position. The loop invariants here is, for each interation, $A[1,…,j]$ consist of the elements originally in $A[1,…,j-1]$, but in sorted order.
  3. Termination: WHen the loop terminates, $j>A.length=n$. Substituing $n+1$ for $j$(Note that referring $j+1$ is just for convenience of describing the process, $A[j+1]$ does not exist), and $A[1,…,n]$ consits of the elements originally in $A[1,…n]$, but in sorted order.

Quicksort

Quicksort applies the divide-and-conquer paradigm.

  1. Divide: Partition (rearrange) the array $A[p,…,r]$ into two (possibly empty) subarrays $A[p,…,q-1]$ and $A[q+1,…,r]$. Each element of $A[p,…q-1]$ is less than or equal to $A[q]$ while each element of $A[q+1,…,r]$ is greater than or equal to $A[q]$.
  2. Conquer: Sort the two subarrays $A[p,…,q-1]$ and $A[q+1,…,r]$ by recursively calling quicksort.

Discription and Correctness

Quicksort is a divide and conquer algorithm. The partition function selects an element $x=A[r]$ as a pivot and rearrange the array so that elements less than $x$ are placed at its left side, while elements greater than $x$ are placed at the right side. The partion process is also a incremental approach, of which correctness can be proved if it has the loop inviants. During each loop, value in $A[i+1,…,j-1]$ are all greater than $x$ while A[p, …, i] are all less than $x$.
Pic5 here

  1. Initialization: Prior to the first iteration of the loop, $i=p-1$ and $j=p$. Since there are no values between $p$ and $i$ and no values lie between $i+1$ and $j-1$, the condition are satisfied.
  2. Maintenance: Since $i$ only increment when condition are satisfied while $j$ imcrement at each interation, $i$ is always less than $j$. At each loop, if $A[j]$ is greater than $x$, the only action during the loop is to increment $j$. Since $j > i$ and $A[j] > x$, condition satisfied. If $A[j]$ is less or equal to $x$, increment $i$ and $j$ by 1 and exchange $A[i]$ and $A[j]$. Since $A[p,…i-1] < x$ and $A[i]_{new}=A[j] <= x$, the loop invariants can be maintaine. On the other hand, $A[i+1,…j-1] < x$ and $A[j]_{new} = A[(i-1)+1]_{old} > x$, loop invariants maintained.
  3. Termination: At termination where $j=r$, we have, $A[p,…i] <= x$ and $A[i+1, r-1] > x$. By exchange $A[i+1]$ and $A[r]$, we have $A[p,…i] < x$, $ A[i+1] = x$ and $A[i+1, r]> x$. Therefore, the partition function return the index of pivot, and all the elements leftwise are less or equal than the pivot, while all the elements rightwise are greater than the pivot.

A quicksort of array $A$ first put the pivot at the right place and return its index, after that, it divide the array into $A[r,…i-1]$ and $A[i+1,…,p]$ and sort it seperately. Assuming the quicksort algorithm is correct, each sub-problem can be solved. If each value in $A[r,…i-1]$ less than or equal to $A[i]$, each value in $A[i+1,…,p]$ greater than $A[i]$, and the two array are sorted, the array $A[r,…,q]$ is sorted.
